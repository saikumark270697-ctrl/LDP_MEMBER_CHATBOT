⚠️
Big problem found: All 3 Contentful entries in the snapshot are Medicaid state-specific pages (Nevada, Florida, California). The main public website pages — Find a Dentist, Member Portal, Plans — are NOT in Contentful as separate entries. The chatbot's knowledge of those topics comes entirely from the 8 hardcoded seed items, not from real CMS pages.
⚠️
Second problem: Contentful's page content type stores actual body text inside referenced section entries (linked by ID). The pipeline only reads metaTitle + metaDescription from the top-level page — it never fetches the section entries that contain the real paragraphs. So Contentful pages are indexed with only their SEO titles and 1–2 sentence descriptions, not the full content.

Problem 2
Body content lives in section entries (linked by ID). Pipeline must fetch those linked entries to get real page text.
Problem 3
inspect_content.py uses no pagination — misses entries beyond first 100. Already fixed in data_pipeline.py.
Problem 4
Home link is dead code in PUBLIC_LINKS — no trigger ever adds it. Either remove it or add a homepage trigger.
