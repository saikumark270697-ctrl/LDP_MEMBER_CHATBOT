📥 How data gets INTO Pinecone
1
Read Contentful CMS
data_pipeline.py connects to the Liberty Dental Contentful space and fetches 4 content types with a 30-second timeout per page.
content_type=section, accordion, page, card
2
Extract plain text
Rich text fields (formatted paragraphs, headings, bullet lists) are recursively converted to plain strings. Empty or very short entries are skipped.
extract_rich_text(node)
3
Convert text to vectors
The all-MiniLM-L6-v2 model converts each document's text into 384 numbers (a vector). Similar meanings produce similar vectors — this is what enables semantic search.
SentenceTransformer.encode()
4
Store in Pinecone
Each vector is stored in Pinecone alongside the original text (as metadata). 496 documents are uploaded in batches of 50.
index.upsert(vectors=records)
💬 How a question gets answered
1
User types a question
The question enters the routing engine. Three checks happen before Pinecone is ever called.
2
3 routing checks (no AI)
Personal data words → blocked instantly. "What can you do?" → hardcoded capabilities list. Everything else passes to Pinecone.
has_restricted_request() · is_capabilities_request()
3
Embed the question
The same all-MiniLM-L6-v2 model converts the question into a 384-number vector so it can be compared with stored document vectors.
4
Search Pinecone (semantic)
Pinecone finds the 6 most similar document vectors. If the best match scores below 0.45 out of 1.0, the bot says it can't find an answer.
top_k=6 · score ≥ 0.45
5
Llama 3.3 70B generates answer
The matched document text is sent to Llama as context. Llama can ONLY answer from that context — it cannot add information from outside it.
max_tokens=600 · temp=0.3
6
Links added by keyword scan
The answer text is scanned for words like "dentist", "app", "portal", "broker". Matching links from the 13-link dictionary are attached below the answer.

🔍 Why "semantic" search, not keyword search
ℹ️ Keyword search only finds exact word matches. Semantic search understands meaning — so "tooth pain" finds the "Teledentistry & Emergencies" document even though neither word appears in the document title.

🛡️ 3 Routing Guards — checked before Pinecone is ever called
Guard 1 — Blocked
Personal data request
Instantly redirects to Member Portal. No AI involved.
"What is my member ID?" "Check my claim" "My SSN" "My benefits"
Guard 2 — Hardcoded
Capabilities question
Returns fixed list of what the bot can help with.
"What can you do?" "How can you help me?" "Who are you?"
Guard 3 — AI search
All other questions
Goes to Pinecone → Llama. Answer from CMS content only.
"Find a dentist" "How do I download the app?" "What plans do you offer?"
✅
Why no personal data ever comes out: Contentful is a public website CMS — it stores page copy, FAQs, and plan descriptions, not member records or account data. Even if someone asked for personal data, there is nothing personal in the 496 indexed documents for Llama to return. Guard 1 blocks the question anyway, as a second layer.
