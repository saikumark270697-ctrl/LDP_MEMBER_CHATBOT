🦷 Will Data Come from Contentful? — Full Strategy

What content types exist · What text is available · The right way to fetch it · Diagnostic step first

Direct Answer
Yes — But You Need to Fetch the RIGHT Content Types
✅
Yes, all data can come from Contentful. The content is there — it's just stored in a nested structure. The current pipeline only reads top-level page entries (3 entries, just SEO titles + descriptions). The actual text — paragraphs, bullet points, FAQs, articles — is inside section entries that are linked from pages. We need to fetch section entries directly.
⚠️
Before implementing anything, we need to run a diagnostic script to see exactly how many section entries exist and what's in them. The CMS could have 50 sections or 500 — we don't know yet. That number will change our approach.

Section 02
3 Ways to Fetch Contentful Data — Which One to Use
✓ Recommended Fetch section entries directly by content type
Use client.entries({'content_type': 'section', 'limit': 200}) with pagination. Each section entry has its own title, description, primaryContent, secondaryContent, and bulletPoints — all readable text. The parentRoute field tells you which page it belongs to. Index each section as its own Pinecone document.
PROS
  • Direct — no nested resolution needed
  • Gets ALL text: paragraphs, bullets, FAQs
  • parentRoute gives page context
  • Works cleanly with pagination
CONS
  • Need separate fetch for accordions/cards
  • Some sections may be shared across pages
~ Alternative Fetch pages with include=2 (Contentful link resolution)
Contentful SDK has an include parameter that auto-resolves linked entries up to N levels deep. With {'content_type': 'page', 'include': 2}, the sections field returns actual section objects instead of link IDs. Traverse them to extract text.
PROS
  • Page title + all section text in one call
  • Keeps page → section relationship clear
CONS
  • More complex to traverse nested objects
  • Only fetches sections linked to pages (may miss orphan sections)
  • Harder to paginate correctly
✗ Current (broken) Fetch page entries, read metaTitle + metaDescription only
What the current data_pipeline.py does. Gets only SEO titles and 1–2 sentence descriptions. The sections field returns link objects (IDs only) which are useless for text extraction.

Section 04 — Do This First
Run This Diagnostic Before Any Code Changes
🔍
Run this script to see exactly what's in your Contentful space — how many sections, what text fields are populated, whether accordion/FAQ entries exist. This takes 2 minutes and tells us everything before we write a single line of pipeline code.
# diagnostic_contentful.py — run this first, don't implement yet import os, contentful from dotenv import load_dotenv load_dotenv() client = contentful.Client( os.getenv("CONTENTFUL_SPACE_ID"), os.getenv("CONTENTFUL_ACCESS_TOKEN"), environment=os.getenv("ENVIRONMENT") ) # Count every content type for ct in ["page", "section", "accordion", "newsletters", "memberTransfer"]: entries = client.entries({'content_type': ct, 'limit': 1}) total = entries.total if hasattr(entries, 'total') else '?' print(f" {ct}: {total} entries") # Preview 3 section entries — see what text fields are populated print("\n--- SECTION PREVIEW ---") sections = client.entries({'content_type': 'section', 'limit': 3}) for s in sections: f = s.fields() print(f"\nTitle: {f.get('title', 'N/A')}") print(f"ParentRoute: {f.get('parentRoute', 'N/A')}") print(f"Has description: {bool(f.get('description'))}") print(f"Has primaryContent: {bool(f.get('primaryContent'))}") print(f"BulletPoints count: {len(f.get('bulletPoints', []))}") print(f"Accordions count: {len(f.get('accordians', []))}") # Preview 3 accordion entries if they exist print("\n--- ACCORDION (FAQ) PREVIEW ---") try: accordions = client.entries({'content_type': 'accordion', 'limit': 3}) for a in accordions: f = a.fields() print(f"Q: {f.get('title') or f.get('question', 'N/A')}") print(f"Has answer: {bool(f.get('answer') or f.get('description'))}") except: print("No accordion entries found")
